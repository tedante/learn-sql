# Sesi 6: Pembersihan & Transformasi Lanjut

**Target Sesi:** Siswa mampu melakukan recode variabel, menangani NULL secara sistematis, membersihkan format data teks/angka/tanggal, dan menulis query analitik bertahap menggunakan CTE.

---

## 1. Pengkodean Ulang dengan Logika Kondisional (CASE WHEN)

Dalam penelitian, kita sering perlu membuat variabel turunan: kategori usia, kategori pendapatan, atau label status tertentu. Di Excel ini mirip IF bertingkat, di SQL kita pakai `CASE WHEN`.

### Contoh 1: Recode Umur ke Kategori Dasar

```sql
SELECT
    nama,
    umur,
    CASE
        WHEN umur < 25 THEN 'Muda'
        WHEN umur BETWEEN 25 AND 44 THEN 'Dewasa Awal'
        WHEN umur BETWEEN 45 AND 59 THEN 'Dewasa Akhir'
        ELSE 'Senior'
    END AS kategori_usia
FROM responden
LIMIT 20;
```

### Contoh 2: Recode Pendapatan ke Status Ekonomi

```sql
SELECT
    nama,
    pendapatan,
    CASE
        WHEN pendapatan < 5000000 THEN 'Rendah'
        WHEN pendapatan BETWEEN 5000000 AND 15000000 THEN 'Menengah'
        ELSE 'Tinggi'
    END AS status_ekonomi
FROM responden
ORDER BY pendapatan DESC
LIMIT 20;
```

### Contoh 3: CASE WHEN di Dalam Agregasi

```sql
SELECT
    wilayah,
    COUNT(*) AS total_responden,
    SUM(CASE WHEN status_pernikahan THEN 1 ELSE 0 END) AS jumlah_menikah,
    SUM(CASE WHEN NOT status_pernikahan THEN 1 ELSE 0 END) AS jumlah_belum_menikah
FROM responden
GROUP BY wilayah
ORDER BY total_responden DESC;
```

**Kapan teknik ini penting:**

1. Saat coding data survei ke kategori analisis.
2. Saat membuat indikator biner (0/1) untuk statistik lanjutan.
3. Saat ingin ringkasan tanpa membuat kolom fisik baru di tabel.

## 2. Menangani Data yang Hilang (NULL Handling)

Data `NULL` bisa menurunkan kualitas analisis jika tidak ditangani jelas. Fungsi utama yang sering dipakai:

1. `COALESCE` untuk memberi nilai fallback.
2. `NULLIF` untuk mengubah nilai tertentu menjadi NULL.
3. `CASE` untuk strategi imputasi kondisional.

### Contoh 1: COALESCE untuk Teks

```sql
SELECT
    nama,
    COALESCE(email, 'tidak_tersedia@example.com') AS email_bersih,
    COALESCE(pekerjaan, 'Belum Bekerja') AS pekerjaan_bersih
FROM responden
LIMIT 20;
```

### Contoh 2: COALESCE untuk Numerik

```sql
SELECT
    wilayah,
    ROUND(AVG(COALESCE(pendapatan, 0)), 0) AS rata_dengan_default_0,
    ROUND(AVG(pendapatan), 0) AS rata_default_sql
FROM responden
GROUP BY wilayah
ORDER BY wilayah;
```

**Catatan:** `AVG(pendapatan)` default SQL akan mengabaikan NULL. Jika Anda memakai `COALESCE(..., 0)`, interpretasi rata-rata bisa berubah. Gunakan dengan sadar.

### Contoh 3: NULLIF untuk Nilai Placeholder

```sql
-- Misal ada data email kosong string '' (bukan NULL), ubah dulu jadi NULL
SELECT
    nama,
    COALESCE(NULLIF(email, ''), 'tidak_tersedia@example.com') AS email_final
FROM responden
LIMIT 20;
```

## 3. Merapikan Teks dan Mengubah Tipe Data (String + Casting)

Data lapangan sering memiliki isu format: kapitalisasi tidak konsisten, spasi berlebih, atau tipe data salah baca.

### Fungsi String yang Sering Dipakai

1. `UPPER` / `LOWER` untuk standarisasi huruf.
2. `TRIM` untuk menghapus spasi awal/akhir.
3. `INITCAP` untuk kapitalisasi awal tiap kata.
4. `LENGTH` untuk cek panjang karakter.
5. `SUBSTRING` untuk potongan teks.

### Contoh 1: Standarisasi Nama dan Wilayah

```sql
SELECT
    nama,
    INITCAP(TRIM(nama)) AS nama_standar,
    UPPER(TRIM(wilayah)) AS wilayah_standar
FROM responden
LIMIT 20;
```

### Contoh 2: Audit Panjang Email

```sql
SELECT
    nama,
    email,
    LENGTH(COALESCE(email, '')) AS panjang_email
FROM responden
ORDER BY panjang_email DESC
LIMIT 20;
```

### Contoh 3: Casting Numerik dan Tanggal

```sql
SELECT
    nama,
    pendapatan,
    pendapatan::TEXT AS pendapatan_teks,
    CAST(umur AS NUMERIC(10,2)) AS umur_decimal,
    tanggal_survey,
    CAST(tanggal_survey AS TEXT) AS tanggal_teks
FROM responden
LIMIT 20;
```

### Contoh 4: Membuat Label Ringkas

```sql
SELECT
    nama,
    wilayah,
    CONCAT(SUBSTRING(INITCAP(nama), 1, 10), ' - ', UPPER(TRIM(wilayah))) AS label_responden
FROM responden
LIMIT 20;
```

## 4. Menulis Query Bertahap dengan CTE (WITH Clause)

CTE membantu memecah query besar menjadi langkah kecil yang mudah dibaca dan diaudit.

### Contoh 1: CTE Sederhana untuk Pembersihan Dasar

```sql
WITH data_bersih AS (
    SELECT
        id,
        INITCAP(TRIM(nama)) AS nama_bersih,
        COALESCE(pendapatan, 0) AS pendapatan_bersih,
        TRIM(wilayah) AS wilayah_bersih,
        umur
    FROM responden
    WHERE umur IS NOT NULL
)
SELECT
    wilayah_bersih,
    COUNT(*) AS jumlah,
    ROUND(AVG(pendapatan_bersih), 0) AS rata_pendapatan
FROM data_bersih
GROUP BY wilayah_bersih
ORDER BY rata_pendapatan DESC;
```

### Contoh 2: CTE Bertingkat (2 Tahap)

```sql
WITH base AS (
    SELECT
        wilayah,
        umur,
        pendapatan,
        CASE
            WHEN pendapatan < 5000000 THEN 'Rendah'
            WHEN pendapatan <= 15000000 THEN 'Menengah'
            ELSE 'Tinggi'
        END AS status_ekonomi
    FROM responden
),
ringkasan AS (
    SELECT
        wilayah,
        status_ekonomi,
        COUNT(*) AS jumlah,
        ROUND(AVG(umur), 1) AS rata_umur,
        ROUND(AVG(pendapatan), 0) AS rata_pendapatan
    FROM base
    GROUP BY wilayah, status_ekonomi
)
SELECT *
FROM ringkasan
WHERE jumlah >= 5
ORDER BY rata_pendapatan DESC;
```

### Contoh 3: CTE untuk Ranking Wilayah

```sql
WITH wilayah_avg AS (
    SELECT
        wilayah,
        ROUND(AVG(pendapatan), 0) AS avg_pendapatan,
        COUNT(*) AS n
    FROM responden
    GROUP BY wilayah
    HAVING COUNT(*) >= 50
)
SELECT
    wilayah,
    avg_pendapatan,
    n,
    DENSE_RANK() OVER (ORDER BY avg_pendapatan DESC) AS peringkat
FROM wilayah_avg
ORDER BY peringkat;
```

## 5. Alur Praktis Data Cleaning (Checklist Cepat)

1. **Audit data mentah**: cek NULL, duplikasi, distribusi nilai aneh.
2. **Standarisasi format**: teks (`TRIM`, `INITCAP`, `UPPER`), tipe (`CAST`).
3. **Recode variabel**: `CASE WHEN` untuk kategori analisis.
4. **Simpan transformasi bertahap**: gunakan CTE agar query mudah diaudit.
5. **Validasi output**: bandingkan `COUNT(*)` sebelum/sesudah transformasi.

---

## Latihan Mandiri Sesi 6

1. **Recode Ekonomi:** Buat kolom `status_ekonomi` dari `pendapatan` dengan 3 kategori (Rendah, Menengah, Tinggi).
2. **Recode Umur:** Buat kolom `kategori_usia` dengan 4 kelompok (Muda, Dewasa Awal, Dewasa Akhir, Senior).
3. **Audit NULL Teks:** Tampilkan `nama`, lalu isi `email` kosong/NULL dengan fallback yang rapi.
4. **Cleaning Teks:** Standarkan `nama` dengan `INITCAP(TRIM(...))` dan `wilayah` dengan `UPPER(TRIM(...))`.
5. **Casting:** Tampilkan `pendapatan` dalam bentuk teks serta `umur` dalam format numerik 2 desimal.
6. **CASE + GROUP BY:** Tampilkan jumlah responden per kategori ekonomi di tiap wilayah.
7. **CTE Dua Tahap:** Buat CTE tahap 1 untuk data bersih, tahap 2 untuk ringkasan rata-rata pendapatan per wilayah. Filter hanya wilayah dengan rata-rata > 10 juta.
8. **CTE + Ranking:** Ranking 5 wilayah dengan rata-rata pendapatan tertinggi (minimal 50 responden per wilayah).
