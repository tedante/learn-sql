## 1. Demo Teaser: Kekuatan SQL (The "Aha!" Moment)

Sebelum belajar kodenya, mari kita lihat hasil akhirnya. Bayangkan Anda memiliki 100.000 baris data penelitian. Di Excel, Anda mungkin butuh banyak klik untuk memfilter dan menghitung ringkasan data. Di SQL, kita cukup menulis beberapa kalimat query:

```sql
-- Contoh 1: Rata-rata umur responden per wilayah
SELECT wilayah, ROUND(AVG(umur), 1) AS rata_rata_umur
FROM responden
GROUP BY wilayah
ORDER BY rata_rata_umur DESC;

-- Contoh 2: Jumlah responden menikah vs belum menikah
SELECT
	CASE
		WHEN status_pernikahan THEN 'Menikah'
		ELSE 'Belum Menikah'
	END AS status,
	COUNT(*) AS jumlah_responden
FROM responden
GROUP BY status_pernikahan;

-- Contoh 3: Top 10 pekerjaan dengan pendapatan rata-rata tertinggi
SELECT pekerjaan, ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan
FROM responden
GROUP BY pekerjaan
ORDER BY rata_rata_pendapatan DESC
LIMIT 10;
```

**Pesan Utama:** SQL bukan tentang "mengetik kode", tapi tentang **efisiensi**. Apa yang butuh 5 menit di Excel, bisa selesai dalam 1 detik di SQL.

## 2. Mengapa SQL untuk Peneliti? (The "Why")

Sebagai peneliti yang terbiasa dengan Excel atau Stata, Anda mungkin bertanya: _"Kenapa saya harus belajar coding jika Excel sudah cukup?"_. Tabel di bawah ini merangkum perbedaan mendasar yang akan memengaruhi kualitas riset Anda:

| Fitur                 | Excel / Spreadsheet                                                 | Stata (.dta)                                                         | SQL (PostgreSQL)                                                                                                |
| :-------------------- | :------------------------------------------------------------------ | :------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------- |
| **Kapasitas Data**    | Terbatas (~1 juta baris). Sering _crash_ jika file terlalu besar.   | Terbatas pada memori RAM komputer Anda.                              | **Hampir tak terbatas.** Mampu mengolah puluhan juta baris dengan stabil.                                       |
| **Integritas Data**   | Sel bisa diubah manual secara tidak sengaja (Raw data rawan rusak). | Data lebih aman, tapi sulit untuk kolaborasi banyak orang sekaligus. | **Sangat Aman.** Data "terkunci" oleh aturan tipe data. Tidak bisa asal ketik di sel.                           |
| **Relasi Data**       | Menggunakan `VLOOKUP`/`XLOOKUP` yang membuat file berat dan lambat. | Menggunakan `merge`, namun terbatas pada penggabungan file statis.   | **Sangat Efisien.** Dirancang khusus untuk menghubungkan banyak tabel (Join) tanpa membebani memori.            |
| **Reproduksibilitas** | Sulit melacak langkah klik-klik atau filter yang pernah dilakukan.  | Bagus dengan `.do` file.                                             | **Sangat Bagus.** Script SQL adalah catatan langkah medis data yang jujur dan bisa dijalankan ulang kapan saja. |
| **Otomasi**           | Harus buka file satu per satu untuk memperbarui data.               | Harus menjalankan script secara manual pada file lokal.              | **Otomatis.** Data bisa diperbarui secara _real-time_ dari berbagai sumber sekaligus.                           |

**Kesimpulan:**
SQL bukan pengganti Stata atau Excel, melainkan **fondasi**. Anda menggunakan SQL untuk mengambil, membersihkan, dan meringkas data besar (Data Wrangling), kemudian hasilnya bisa Anda tarik ke Stata atau Excel untuk analisis statistik akhir.

### Tips:

_"Bayangkan SQL sebagai asisten pribadi yang bertugas masuk ke gudang data yang sangat berantakan dan luas, lalu membawakan tabel yang rapi dan siap pakai tepat ke meja kerja Anda (Stata/Excel)."_

## 3. Bergeser dari Spreadsheet ke Database (Mental Model)

Jangan bayangkan database sebagai sesuatu yang asing. Gunakan padanan istilah ini:

| Istilah SQL        | Padanan di Excel/Stata | Keterangan                                          |
| :----------------- | :--------------------- | :-------------------------------------------------- |
| **Database**       | Folder Proyek          | Wadah besar untuk semua data riset Anda.            |
| **Table**          | Sheet / File .dta      | Satu kumpulan data (misal: data responden).         |
| **Row / Record**   | Baris / Observasi      | Satu individu atau satu kejadian.                   |
| **Column / Field** | Kolom / Variabel       | Atribut yang diukur (Umur, Jenis Kelamin).          |
| **Schema**         | Daftar Isi             | Peta yang menunjukkan tabel apa saja yang tersedia. |

---

## 4. Mengenal Alat Kerja (The Tools)

Untuk bekerja dengan data, kita menggunakan dua komponen:

1.  **PostgreSQL (The Engine):** Mesin yang menyimpan data. Ibaratnya seperti mesin mobil.
2.  **DBeaver / pgAdmin (The Interface):** Aplikasi tempat kita mengetik kode dan melihat tabel. Ibaratnya seperti kemudi dan dasbor mobil.

> **Tugas:** Pastikan Anda bisa terhubung (_Connect_) ke database menggunakan kredensial yang diberikan oleh instruktur.

---

## 5. Logika Deklaratif: Berbicara dengan Data

Ini adalah perbedaan cara berpikir yang paling mendasar:

- **Excel (Prosedural):** "Pergi ke kolom B, klik Filter, pilih Jakarta."
- **SQL (Deklaratif):** "Tampilkan nama dari tabel responden yang berasal dari Jakarta."

**Perintah Pertama Anda:**

```sql
SELECT * FROM responden;
```

- `SELECT`: Instruksi untuk menampilkan data.
- `*`: Simbol untuk "Semua Kolom".
- `FROM responden`: Menentukan tabel sumbernya.
- `;`: Tanda titik koma wajib ada sebagai penutup instruksi.
