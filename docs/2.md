# Sesi 2: Membangun & Mengelola Data

**Target Sesi:** Siswa mampu membuat struktur tabel sendiri dan melakukan manipulasi data (Input, Edit, Hapus) dengan aman.

---

## 1. Mendesain Tabel: Menentukan Aturan Main (Data Types)

Di Excel, Anda bisa mengetik apa saja di sel mana saja. Di SQL, kita harus menentukan "tipe" data sejak awal. Ini adalah kunci agar data penelitian Anda tetap bersih dan valid.

| Tipe Data SQL         | Kegunaan                          | Contoh Data                      |
| :-------------------- | :-------------------------------- | :------------------------------- |
| `INTEGER`             | Angka bulat.                      | Umur, Jumlah Anak, ID.           |
| `NUMERIC` / `DECIMAL` | Angka desimal presisi (Keuangan). | Pendapatan, Berat Badan, Tinggi. |
| `VARCHAR(n)`          | Teks dengan batas karakter.       | Nama, Alamat, Email.             |
| `TEXT`                | Teks panjang tanpa batas.         | Catatan wawancara, Komentar.     |
| `BOOLEAN`             | Pilihan Benar/Salah (Ya/Tidak).   | Status Menikah (True/False).     |
| `DATE`                | Tanggal.                          | Tanggal Survey, Tanggal Lahir.   |

---

## 2. Membuat Tabel (CREATE TABLE)

Inilah cara kita membuat "rumah" untuk data kita. Bayangkan ini seperti menyiapkan _header_ kolom dan format sel di Excel sekaligus.

```sql
CREATE TABLE responden_baru (
    id SERIAL PRIMARY KEY,        -- ID otomatis unik
    nama VARCHAR(100) NOT NULL,   -- Nama tidak boleh kosong
    umur INTEGER CHECK (umur > 0), -- Umur harus angka positif
    kota VARCHAR(50),
    tanggal_gabung DATE DEFAULT CURRENT_DATE
);
```

---

## 3. Mengisi & Mengelola Data (DML)

Sekarang, mari kita belajar cara mengisi, mengubah, dan menghapus data secara manual. Ini sangat berguna jika Anda menemukan kesalahan input pada satu atau dua responden.

### A. Memasukkan Data (INSERT)

```sql
INSERT INTO responden_baru (nama, umur, kota)
VALUES ('Rina Sari', 28, 'Bandung');
```

### B. Mengubah Data (UPDATE)

**Hati-hati!** Selalu gunakan `WHERE`. Tanpa `WHERE`, Anda akan mengubah **seluruh** baris di tabel tersebut.

```sql
SELECT id, nama, kota
FROM responden_baru
WHERE id = 1; -- Cek dulu target baris

UPDATE responden_baru
SET kota = 'Jakarta'
WHERE id = 1
RETURNING id, nama, kota; -- Verifikasi hasil perubahan
```

### C. Menghapus Data (DELETE)

Sama seperti Update, jangan lupa `WHERE`.

```sql
DELETE FROM responden_baru
WHERE id = 1; -- Menghapus responden dengan ID tertentu
```

---

## 4. Keamanan Data: Transaksi (COMMIT & ROLLBACK)

Di SQL, tidak ada tombol "Ctrl + Z" (Undo) yang instan. Namun, PostgreSQL punya fitur **Transaction**. Ini seperti membuat "Draft" sebelum benar-benar menyimpannya secara permanen.

**Catatan penting:** jika Anda memakai DBeaver/pgAdmin, pastikan mode auto-commit nonaktif saat latihan transaksi. Jika auto-commit aktif, perubahan bisa langsung permanen dan `ROLLBACK` tidak berdampak.

```sql
BEGIN; -- Mulai mode draft

DELETE FROM responden_baru; -- Oops, saya tidak sengaja menghapus semua!

ROLLBACK; -- BATALKAN! Data kembali seperti semula.
-- Atau gunakan COMMIT; jika Anda yakin ingin simpan permanen.
```
