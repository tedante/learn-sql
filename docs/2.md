**Target Sesi:** Siswa mampu membuat struktur tabel sendiri dan melakukan manipulasi data (Input, Edit, Hapus) dengan aman.

## 1. Mendesain Tabel: Menentukan Aturan Main (Data Types & Constraints)

Di Excel, Anda bisa mengetik apa saja di sel mana saja. Di SQL, kita harus menentukan "tipe" data sejak awal. Ini adalah kunci agar data penelitian Anda tetap bersih dan valid.

### Tipe Data SQL

| Tipe Data SQL              | Kegunaan                          | Contoh Data                      |
| :------------------------- | :-------------------------------- | :------------------------------- |
| `INTEGER`                  | Angka bulat.                      | Umur, Jumlah Anak, ID.           |
| `NUMERIC(p,s)` / `DECIMAL` | Angka desimal presisi (Keuangan). | Pendapatan, Berat Badan, Tinggi. |
| `VARCHAR(n)`               | Teks dengan batas karakter.       | Nama, Alamat, Email.             |
| `TEXT`                     | Teks panjang tanpa batas.         | Catatan wawancara, Komentar.     |
| `BOOLEAN`                  | Pilihan Benar/Salah (Ya/Tidak).   | Status Menikah (True/False).     |
| `DATE`                     | Tanggal.                          | Tanggal Survey, Tanggal Lahir.   |
| `TIMESTAMP`                | Tanggal dan waktu detail.         | Waktu pencatat, Created at.      |

### Aturan Kolom (Constraints)

Setelah memilih tipe, kita bisa tambahkan aturan agar data semakin valid:

| Constraint    | Arti                                               | Contoh                                      |
| :------------ | :------------------------------------------------- | :------------------------------------------ |
| `PRIMARY KEY` | ID unik yang tidak boleh duplikat atau kosong.     | `id SERIAL PRIMARY KEY`                     |
| `NOT NULL`    | Kolom wajib diisi, tidak boleh kosong.             | `nama VARCHAR(100) NOT NULL`                |
| `UNIQUE`      | Nilai harus unik (tidak boleh duplikat).           | `email VARCHAR(100) UNIQUE`                 |
| `DEFAULT`     | Nilai otomatis jika tidak dikasih input.           | `status VARCHAR(20) DEFAULT 'aktif'`        |
| `CHECK`       | Nilai harus memenuhi kondisi tertentu.             | `umur INTEGER CHECK (umur >= 0)`            |
| `FOREIGN KEY` | Link ke ID di tabel lain (akan dipelajari sesi 5). | `id_wilayah INTEGER REFERENCES wilayah(id)` |

## 2. Membuat Tabel (CREATE TABLE)

Inilah cara kita membuat "rumah" untuk data kita. Bayangkan ini seperti menyiapkan _header_ kolom dan format sel di Excel sekaligus.

### Contoh 1: Tabel Sederhana

```sql
CREATE TABLE responden_baru (
    id SERIAL PRIMARY KEY,        -- ID otomatis unik
    nama VARCHAR(100) NOT NULL,   -- Nama tidak boleh kosong
    umur INTEGER CHECK (umur > 0), -- Umur harus angka positif
    kota VARCHAR(50),
    tanggal_gabung DATE DEFAULT CURRENT_DATE
);
```

### Contoh 2: Tabel dengan Constraint UNIQUE

Gunakan `UNIQUE` jika ada kolom yang tidak boleh duplikat (seperti email atau nomor identitas).

```sql
CREATE TABLE pengguna (
    id SERIAL PRIMARY KEY,
    nama_pengguna VARCHAR(50) NOT NULL UNIQUE,  -- Username tidak boleh kembar
    email VARCHAR(100) NOT NULL UNIQUE,         -- Email juga tidak boleh kembar
    tanggal_daftar DATE DEFAULT CURRENT_DATE
);
```

### Contoh 3: Tabel dengan Constraint Berlapis

Kita bisa menggabungkan banyak constraint di satu tabel untuk keamanan maksimal.

```sql
CREATE TABLE rekam_medis (
    id_pasien SERIAL PRIMARY KEY,
    nama_pasien VARCHAR(100) NOT NULL,
    nomor_identitas VARCHAR(20) NOT NULL UNIQUE,
    tanggal_lahir DATE NOT NULL,
    berat_badan NUMERIC(5,2) CHECK (berat_badan > 0),
    tinggi_badan NUMERIC(5,2) CHECK (tinggi_badan > 0),
    status_aktif BOOLEAN DEFAULT TRUE,
    tanggal_pemeriksaan DATE DEFAULT CURRENT_DATE
);
```

## 3. Mengisi & Mengelola Data (DML)

Sekarang, mari kita belajar cara mengisi, mengubah, dan menghapus data secara manual. Ini sangat berguna jika Anda menemukan kesalahan input pada satu atau dua responden.

### A. Memasukkan Data (INSERT)

#### Contoh 1: INSERT Baris Tunggal

```sql
INSERT INTO responden_baru (nama, umur, kota)
VALUES ('Rina Sari', 28, 'Bandung');
```

#### Contoh 2: INSERT dengan Verifikasi ID (RETURNING)

Karena `id` adalah `SERIAL` (otomatis), kita pakai `RETURNING` untuk lihat ID yang dihasilkan.

```sql
INSERT INTO responden_baru (nama, umur, kota)
VALUES ('Budi Santoso', 35, 'Jakarta')
RETURNING id, nama, kota;  -- Tampilkan hasil insert termasuk id otomatis
```

#### Contoh 3: INSERT Multiple Rows (Batch)

Daripada insert satu-satu, bisa insert banyak baris sekaligus, lebih cepat.

```sql
INSERT INTO responden_baru (nama, umur, kota)
VALUES
    ('Siti Rahma', 26, 'Medan'),
    ('Ahmad Wijaya', 40, 'Surabaya'),
    ('Dewi Lestari', 32, 'Bandung');
```

#### Contoh 4: INSERT dengan DEFAULT (Kolom tidak perlu diisi)

```sql
INSERT INTO responden_baru (nama, umur, kota)
VALUES ('Hani Kusuma', 29, 'Yogyakarta');
-- tanggal_gabung otomatis pakai DEFAULT CURRENT_DATE
```

### B. Mengubah Data (UPDATE)

**Hati-hati!** Selalu gunakan `WHERE`. Tanpa `WHERE`, Anda akan mengubah **seluruh** baris di tabel tersebut.

#### Contoh 1: Safety Pattern - Cek Dulu Sebelum UPDATE

Sebagai praktik aman, jalankan `SELECT` dengan `WHERE` yang sama dulu.

```sql
-- Step 1: Cek target baris
SELECT id, nama, kota
FROM responden_baru
WHERE id = 1;

-- Step 2: Jika sudah yakin target benar, baru jalankan UPDATE
UPDATE responden_baru
SET kota = 'Jakarta'
WHERE id = 1
RETURNING id, nama, kota;  -- Verifikasi hasil perubahan
```

#### Contoh 2: UPDATE Satu Kolom dengan Kondisi Sederhana

```sql
UPDATE responden_baru
SET umur = 31
WHERE nama = 'Rina Sari'
RETURNING id, nama, umur;
```

#### Contoh 3: UPDATE Multiple Kolom Sekaligus

```sql
UPDATE responden_baru
SET
    kota = 'Bandung',
    umur = 30
WHERE id = 2
RETURNING id, nama, umur, kota;
```

#### Contoh 4: UPDATE dengan Expression (Perhitungan)

Kita bisa UPDATE dengan nilai yang dihitung dari kolom lain.

```sql
-- Naikkan umur semua responden 1 tahun (contoh ulang tahun masal)
UPDATE responden_baru
SET umur = umur + 1
WHERE kota = 'Jakarta'
RETURNING id, nama, umur;
```

#### Contoh 5: UPDATE dengan Kondisi WHERE Kompleks

```sql
-- Ubah kota responden yang umurnya di atas 30 dan dari Jakarta
UPDATE responden_baru
SET kota = 'Bogor'
WHERE umur > 30 AND kota = 'Jakarta'
RETURNING id, nama, umur, kota;
```

### C. Menghapus Data (DELETE)

Sama seperti Update, **jangan lupa `WHERE`**. Tanpa WHERE, semua baris akan terhapus!

#### Contoh 1: DELETE by ID (Paling Aman)

Menghapus berdasarkan ID adalah cara teraman karena ID adalah unique.

```sql
DELETE FROM responden_baru
WHERE id = 1;
```

#### Contoh 2: DELETE dengan Kondisi WHERE Sederhana

```sql
DELETE FROM responden_baru
WHERE kota = 'Bandung';
```

#### Contoh 3: DELETE dengan IN Clause (Batch Delete)

Menghapus lebih dari satu id sekaligus.

```sql
DELETE FROM responden_baru
WHERE id IN (5, 10, 15); -- Menghapus responden dengan id 5, 10, 15
```

#### Contoh 4: DELETE dengan Kondisi Kompleks

```sql
DELETE FROM responden_baru
WHERE umur < 18 AND kota = 'Jakarta';
```

#### Contoh 5: DELETE dengan Cek Dulu (Safe Pattern)

Hampir selalu, jalankan SELECT dengan WHERE yang sama untuk verifikasi.

```sql
-- Step 1: Lihat data yang akan dihapus
SELECT COUNT(*), id
FROM responden_baru
WHERE umur > 60
GROUP BY id;

-- Step 2: Jika jumlahnya masuk akal, baru jalankan DELETE
DELETE FROM responden_baru
WHERE umur > 60;
```

## 4. Keamanan Data: Transaksi (BEGIN, COMMIT & ROLLBACK)

Di SQL, tidak ada tombol "Ctrl + Z" (Undo) yang instan. Namun, PostgreSQL punya fitur **Transaction**. Ini seperti membuat "Draft" sebelum benar-benar menyimpannya secara permanen.

**Catatan penting:** jika Anda memakai DBeaver/pgAdmin, pastikan mode auto-commit nonaktif saat latihan transaksi. Jika auto-commit aktif, perubahan bisa langsung permanen dan `ROLLBACK` tidak berdampak.

### Contoh 1: Transaction dengan ROLLBACK (Batalkan)

Gunakan `ROLLBACK` jika mau membatalkan perubahan.

```sql
BEGIN; -- Mulai mode draft (transaction started)

DELETE FROM responden_baru WHERE id = 5; -- Hapus satu baris
UPDATE responden_baru SET kota = 'Medan' WHERE id = 3;

ROLLBACK; -- BATALKAN! Semua perubahan di atas dibatalkan, data kembali seperti semula.
```

### Contoh 2: Transaction dengan COMMIT (Simpan Permanen)

Gunakan `COMMIT` jika sudah yakin dengan perubahan.

```sql
BEGIN; -- Mulai mode draft

INSERT INTO responden_baru (nama, umur, kota) VALUES ('Rudi', 45, 'Semarang');
INSERT INTO responden_baru (nama, umur, kota) VALUES ('Lina', 38, 'Medan');

COMMIT; -- Simpan permanen! Kedua INSERT sudah jadi bagian database.
```

### Contoh 3: Simulasi Error - ROLLBACK Otomatis

Jika ada error di tengah transaction, biasanya transaction dibatalkan otomatis.

```sql
BEGIN;

INSERT INTO responden_baru (nama, umur, kota) VALUES ('Eka', 'BUKAN_ANGKA', 'Bali');
-- Error! umur harus INTEGER, bukan teks.
-- Transaction otomatis dibatalkan (ROLLBACK).
```

### Contoh 4: Transaction Multi-Statement (Praktik Kompleks)

Kita bisa combine INSERT, UPDATE, DELETE dalam satu transaction.

```sql
BEGIN;

INSERT INTO responden_baru (nama, umur, kota)
VALUES ('Fadli', 28, 'Palembang');

UPDATE responden_baru
SET umur = umur + 1
WHERE kota = 'Jakarta';

DELETE FROM responden_baru
WHERE umur > 70;

COMMIT; -- Semua 3 operasi disimpan sekaligus
```

1. **Buat Tabel Baru:** Buat tabel bernama `survey_kesehatan` dengan kolom: `id` (PRIMARY KEY), `nama_pasien` (NOT NULL), `berat_badan` (`NUMERIC(5,2)`), `tinggi_badan` (`NUMERIC(5,2)`), `perokok` (boolean), dan `tanggal_pemeriksaan` (DATE dengan DEFAULT CURRENT_DATE).

2. **Input Data - Single Row:** Masukkan 1 pasien dengan `RETURNING` untuk lihat id otomatis.

3. **Input Data - Multiple Rows:** Masukkan 4 pasien fiktif dalam satu perintah INSERT.

4. **Lihat Semua Data:** Jalankan `SELECT * FROM survey_kesehatan;`

5. **Koreksi Data:**
   - Cek dulu responden yang ingin diubah dengan `SELECT`.
   - Gunakan `UPDATE` untuk mengubah berat badan salah satu pasien dengan `RETURNING` untuk verifikasi.

6. **Multi-Column Update:** Update 2 kolom sekaligus (misal: berat dan perokok) untuk 1 pasien tertentu.

7. **Hapus Data:**
   - Cek dulu dengan SELECT berapa banyak baris dengan kondisi tertentu (misal: perokok = TRUE).
   - DELETE baris tersebut.

8. **Simulasi Error dan Transaction:**
   - Coba INSERT teks ke dalam kolom `berat_badan`. Amati pesan kesalahannya.
   - Gunakan `BEGIN;`, lakukan beberapa INSERT/UPDATE, lalu cek dengan `SELECT`, akhirnya `ROLLBACK;`. Verifikasi data kembali ke semula.
   - Ulangi dengan `COMMIT` untuk simpan permanen.

## Cheat Sheet: Safety Pattern untuk DML

Untuk menghindari kesalahan fatal, selalu ikuti pola ini sebelum UPDATE atau DELETE:

```sql
-- Step 1: SELECT untuk verifikasi target (MANDATORY!)
SELECT * FROM [tabel]
WHERE [kondisi_yang_sama];

-- Step 2: Pastikan hasil SELECT sesuai ekspektasi.
-- Jika iya, baru jalankan UPDATE atau DELETE.

-- Step 3: UPDATE atau DELETE dengan RETURNING
UPDATE [tabel]
SET [kolom] = [nilai]
WHERE [kondisi_yang_sama]
RETURNING *;  -- atau RETURNING id, nama, dst

-- Atau:
DELETE FROM [tabel]
WHERE [kondisi_yang_sama];
```

### Tips:

_"Dalam manajemen data, perintah `UPDATE` dan `DELETE` adalah pedang bermata dua. Selalu lakukan `SELECT` terlebih dahulu dengan kriteria `WHERE` yang sama untuk memastikan Anda membidik baris yang tepat sebelum menekan tombol eksekusi. Jika ragu, gunakan transaction dengan BEGIN, cek hasil, baru COMMIT."_
