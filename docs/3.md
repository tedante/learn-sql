# Sesi 3: Penarikan Data Dasar (Data Discovery)

**Target Sesi:** Siswa mampu melakukan eksplorasi data secara mandiri, menyaring observasi dengan kriteria kompleks, dan melakukan audit kualitas data (data cleaning awal).

---

## 1. Memilih dan Memberi Nama Variabel (SELECT & AS)

Dalam SQL, kita tidak hanya mengambil kolom, tapi bisa memodifikasi tampilannya agar sesuai dengan label laporan penelitian.

- **SELECT:** Menentukan kolom mana yang akan ditarik.
- **AS (Alias):** Memberikan nama sementara pada kolom agar lebih informatif.

```sql
-- Contoh 1: Mengambil variabel spesifik dan mengganti nama kolom output
SELECT
    nama AS nama_responden,
    wilayah AS kota_domisili,
    umur,
    pendapatan AS gaji_bulanan,
    (pendapatan * 12) AS estimasi_pendapatan_tahunan,
    status_pernikahan AS sudah_menikah
FROM responden;

-- Contoh 2: Membatasi hasil agar mudah dibaca saat eksplorasi awal
SELECT id, nama, pekerjaan, wilayah
FROM responden
LIMIT 20;
```

---

## 2. Penyaringan Baris Tingkat Lanjut (WHERE Clauses)

Penyaringan adalah bagian terpenting dalam riset untuk menentukan **Sampel**.

### A. Operator Logika Kombinasi (AND, OR, NOT)

Seringkali kita butuh kriteria berlapis. Gunakan tanda kurung `()` untuk memastikan urutan logika benar (seperti dalam matematika).

```sql
-- Skenario 1: Responden di Jakarta usia > 30 dan sudah menikah
SELECT id, nama, wilayah, umur, status_pernikahan
FROM responden
WHERE wilayah = 'Jakarta'
    AND umur > 30
    AND status_pernikahan = TRUE;

-- Skenario 2: Responden di Bandung ATAU Surabaya, tapi bukan usia di bawah 21
SELECT id, nama, wilayah, umur
FROM responden
WHERE wilayah IN ('Bandung', 'Surabaya')
    AND NOT umur < 21;
```

### B. Penyaringan Daftar dan Rentang (IN & BETWEEN)

Daripada menulis banyak `OR`, gunakan `IN`. Untuk rentang angka atau tanggal, gunakan `BETWEEN`.

```sql
-- Mencari responden di kota-kota besar tertentu
SELECT id, nama, wilayah, pekerjaan
FROM responden
WHERE wilayah IN ('Jakarta', 'Surabaya', 'Medan', 'Makassar');

-- Mencari responden dalam kelompok usia produktif (20-40 tahun)
SELECT id, nama, umur
FROM responden
WHERE umur BETWEEN 20 AND 40;

-- Mencari responden yang disurvei dalam 90 hari terakhir
SELECT id, nama, tanggal_survey
FROM responden
WHERE tanggal_survey BETWEEN CURRENT_DATE - INTERVAL '90 days' AND CURRENT_DATE;
```

### C. IS NULL, IS NOT NULL, dan Operator Perbandingan

Saat eksplorasi, kita sering perlu mengecek data kosong dan membuat filter numerik.

```sql
-- Cek data email yang kosong
SELECT id, nama, wilayah
FROM responden
WHERE email IS NULL;

-- Cek data email yang terisi
SELECT id, nama, email
FROM responden
WHERE email IS NOT NULL
LIMIT 10;

-- Filter pendapatan lebih dari 15 juta
SELECT id, nama, pendapatan
FROM responden
WHERE pendapatan > 15000000;
```

## 3. Audit Kualitas Data (IS NULL & DISTINCT)

Sebelum dianalisis, peneliti harus tahu apakah datanya "kotor" atau ada yang bolong (_missing values_).

- **IS NULL:** Mencari data yang tidak diisi.
- **DISTINCT:** Melihat daftar unik dari suatu kategori (misal: ingin tahu ada berapa variasi pekerjaan di database).

```sql
-- Audit 1: Berapa banyak data kosong per kolom penting?
SELECT
        COUNT(*) FILTER (WHERE nama IS NULL) AS nama_kosong,
        COUNT(*) FILTER (WHERE email IS NULL) AS email_kosong,
        COUNT(*) FILTER (WHERE pekerjaan IS NULL) AS pekerjaan_kosong,
        COUNT(*) FILTER (WHERE wilayah IS NULL) AS wilayah_kosong
FROM responden;

-- Audit 2: Daftar wilayah unik
SELECT DISTINCT wilayah
FROM responden
ORDER BY wilayah;

-- Audit 3: Berapa jumlah kategori pekerjaan unik?
SELECT COUNT(DISTINCT pekerjaan) AS jumlah_kategori_pekerjaan
FROM responden;
```

## 4. Pencarian Berdasarkan Pola (LIKE dan ILIKE)

Sangat berguna jika Anda mencari kata kunci tertentu dalam kolom teks (seperti catatan medis atau judul pekerjaan).

- `%` : Mewakili banyak karakter (apapun).
- `_` : Mewakili tepat satu karakter.
- `ILIKE` : Sama seperti `LIKE`, tetapi tidak sensitif huruf besar-kecil.

```sql
-- Mencari semua responden yang bekerja di bidang 'Analyst'
-- (Akan memunculkan: Data Analyst, Financial Analyst, System Analyst)
SELECT nama, pekerjaan FROM responden
WHERE pekerjaan ILIKE '%analyst%';

-- Mencari responden yang nama depannya 'Siti'
SELECT id, nama, wilayah
FROM responden
WHERE nama ILIKE 'siti%';

-- Mencari pekerjaan yang berakhiran 'Manager'
SELECT id, nama, pekerjaan
FROM responden
WHERE pekerjaan ILIKE '%manager';

-- Contoh penggunaan underscore: nama 4 huruf, diawali 'A'
SELECT id, nama
FROM responden
WHERE nama LIKE 'A___%';
```

## 5. Navigasi Hasil (ORDER BY, LIMIT, OFFSET)

Setelah data disaring, kita perlu mengaturnya agar mudah dibaca atau mengambil sampel acak.

- **ORDER BY:** Mengurutkan (Default: `ASC`/Kecil ke Besar. Gunakan `DESC` untuk Besar ke Kecil).
- **LIMIT:** Mengambil _n_ baris pertama.
- **OFFSET:** Melewati sejumlah baris (sering digunakan untuk melihat data secara bertahap).

```sql
-- Contoh 1: 10 responden dengan pendapatan tertinggi di Jakarta
SELECT nama, pendapatan, wilayah
FROM responden
WHERE wilayah = 'Jakarta'
ORDER BY pendapatan DESC
LIMIT 10;

-- Contoh 2: Pagination (halaman 1, 10 baris pertama)
SELECT id, nama, wilayah, tanggal_survey
FROM responden
ORDER BY id ASC
LIMIT 10 OFFSET 0;

-- Contoh 3: Pagination (halaman 2, 10 baris berikutnya)
SELECT id, nama, wilayah, tanggal_survey
FROM responden
ORDER BY id ASC
LIMIT 10 OFFSET 10;

-- Contoh 4: Sampel acak 15 baris
SELECT id, nama, wilayah, pekerjaan
FROM responden
ORDER BY RANDOM()
LIMIT 15;
```

Catatan: untuk pagination yang stabil, selalu gunakan `ORDER BY` pada kolom yang konsisten (misalnya `id`).

## Tabel Ringkasan Operator Discovery

| Kebutuhan Riset           | Operator SQL        | Analogi Excel/Stata          |
| :------------------------ | :------------------ | :--------------------------- |
| **Ambil kolom tertentu**  | `SELECT col1, col2` | `keep var1 var2` (Stata)     |
| **Filter kriteria ganda** | `WHERE ... AND ...` | Multi-level Filter           |
| **Filter daftar pilihan** | `IN ('A', 'B')`     | Filter dengan banyak centang |
| **Filter rentang nilai**  | `BETWEEN x AND y`   | Number Filter: Between       |
| **Cari data bolong**      | `IS NULL`           | Filter: (Blanks)             |
| **Cari data terisi**      | `IS NOT NULL`       | Filter: (Non-Blanks)         |
| **Lihat kategori unik**   | `DISTINCT`          | Remove Duplicates (Visual)   |
| **Cari teks fleksibel**   | `ILIKE '%teks%'`    | Contains (tidak peka huruf)  |
| **Urutkan data**          | `ORDER BY`          | Sort A to Z / Z to A         |
| **Ambil sebagian baris**  | `LIMIT ... OFFSET`  | Lihat data per halaman       |

## Latihan Mandiri Sesi 3 (Deep Dive)

1. **Analisis Sampel:** Tampilkan responden yang bekerja sebagai 'Teacher' atau 'Doctor' di Bandung, lalu urutkan berdasarkan umur dari termuda.
2. **Audit Data:** Tampilkan responden yang memiliki email terisi tetapi pekerjaan kosong (`email IS NOT NULL` dan `pekerjaan IS NULL`).
3. **Proyeksi Sederhana:** Tampilkan `nama`, `pendapatan`, dan kolom baru `pendapatan_setelah_pajak` (pendapatan dikurangi 10%).
4. **Pencarian Spesifik:** Cari semua responden yang namanya mengandung kata 'Agus' dan pekerjaan yang mengandung kata 'Manager' (gunakan `ILIKE`).
5. **Tanggal Survey:** Ambil 20 responden dengan `tanggal_survey` terbaru.
6. **Pagination:** Tampilkan data halaman ke-3 dengan ukuran 15 baris, urut berdasarkan `id`.

### Kunci Jawaban Singkat (Opsional Cek Mandiri)

```sql
-- 1
SELECT id, nama, pekerjaan, wilayah, umur
FROM responden
WHERE wilayah = 'Bandung'
    AND pekerjaan IN ('Teacher', 'Doctor')
ORDER BY umur ASC;

-- 2
SELECT id, nama, email, pekerjaan
FROM responden
WHERE email IS NOT NULL
    AND pekerjaan IS NULL;

-- 3
SELECT
    nama,
    pendapatan,
    ROUND(pendapatan * 0.9, 0) AS pendapatan_setelah_pajak
FROM responden;

-- 4
SELECT id, nama, pekerjaan
FROM responden
WHERE nama ILIKE '%agus%'
    AND pekerjaan ILIKE '%manager%';

-- 5
SELECT id, nama, tanggal_survey
FROM responden
ORDER BY tanggal_survey DESC
LIMIT 20;

-- 6
SELECT id, nama, wilayah
FROM responden
ORDER BY id ASC
LIMIT 15 OFFSET 30;
```

### Tips:

_"Dalam SQL, urutan penulisan sangat penting. Untuk query dasar, ingat peta ini: `SELECT` -> `FROM` -> `WHERE` -> `GROUP BY` -> `HAVING` -> `ORDER BY` -> `LIMIT/OFFSET`. Jika urutan salah, query bisa gagal atau hasilnya keliru."_
