# Sesi 4: Agregasi & Ringkasan (The Pivot Logic)

**Target Sesi:** Siswa mampu melakukan perhitungan statistik deskriptif dan meringkas data berdasarkan kategori tertentu untuk kebutuhan laporan penelitian.

---

## 1. Fungsi Agregasi Dasar (Statistik Deskriptif)

Fungsi agregasi digunakan untuk melakukan perhitungan pada sekumpulan nilai dan menghasilkan satu nilai ringkasan.

| Fungsi            | Kegunaan                        | Contoh Kasus Penulisan                  |
| :---------------- | :------------------------------ | :-------------------------------------- |
| `COUNT()`         | Menghitung jumlah baris/sampel. | Berapa total responden kita?            |
| `COUNT(DISTINCT)` | Menghitung nilai unik.          | Ada berapa kategori pekerjaan?          |
| `SUM()`           | Menjumlahkan nilai angka.       | Berapa total pendapatan seluruh sampel? |
| `AVG()`           | Menghitung rata-rata.           | Berapa rata-rata umur responden?        |
| `MIN()`           | Mencari nilai terendah.         | Berapa umur responden termuda?          |
| `MAX()`           | Mencari nilai tertinggi.        | Berapa pendapatan tertinggi di dataset? |

### Contoh 1: COUNT - Jumlah Total Baris

```sql
SELECT COUNT(*) AS total_responden
FROM responden;
```

### Contoh 2: COUNT(DISTINCT) - Menghitung Kategori Unik

```sql
SELECT COUNT(DISTINCT pekerjaan) AS jumlah_kategori_pekerjaan
FROM responden;
```

### Contoh 3: Multiple Agregasi Sekaligus

```sql
SELECT
    COUNT(*) AS total_sampel,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan,
    MIN(pendapatan) AS pendapatan_terendah,
    MAX(pendapatan) AS pendapatan_tertinggi,
    ROUND(AVG(umur), 1) AS rata_rata_umur
FROM responden;
```

### Contoh 4: SUM untuk Total Penjumlahan

```sql
SELECT
    SUM(pendapatan) AS total_pendapatan_semua,
    COUNT(*) AS jumlah_responden,
    ROUND(SUM(pendapatan) / COUNT(*), 0) AS rata_rata_manual
FROM responden;
```

## 2. Pengelompokan Data (GROUP BY)

Inilah kekuatan utama SQL. `GROUP BY` memberi tahu database untuk mengumpulkan baris yang memiliki nilai yang sama ke dalam satu grup, lalu menjalankan fungsi agregasi pada tiap grup tersebut.

**Analogi:** Seperti membuat Pivot Table di Excel, di mana Anda menaruh "Wilayah" di bagian _Rows_ dan "Pendapatan" di bagian _Values_.

### Contoh 1: GROUP BY Sederhana - Satu Kolom

```sql
-- Skenario: Hitung jumlah responden per wilayah
SELECT
    wilayah,
    COUNT(*) AS jumlah_responden
FROM responden
GROUP BY wilayah
ORDER BY jumlah_responden DESC;
```

### Contoh 2: GROUP BY dengan Multiple Agregasi

```sql
-- Skenario: Menghitung rata-rata dan total pendapatan per wilayah
SELECT
    wilayah,
    COUNT(*) AS jumlah_orang,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan,
    ROUND(SUM(pendapatan), 0) AS total_pendapatan
FROM responden
GROUP BY wilayah
ORDER BY total_pendapatan DESC;
```

### Contoh 3: GROUP BY Dua Kolom

```sql
-- Skenario: Rata-rata pendapatan berdasarkan wilayah dan status pernikahan
SELECT
    wilayah,
    status_pernikahan,
    COUNT(*) AS jumlah,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan
FROM responden
GROUP BY wilayah, status_pernikahan
ORDER BY wilayah, status_pernikahan;
```

### Contoh 4: GROUP BY dengan WHERE Filter Dulu

```sql
-- Skenario: Rata-rata pendapatan per pekerjaan, hanya untuk responden usia 25-45 tahun
SELECT
    pekerjaan,
    COUNT(*) AS jumlah,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan
FROM responden
WHERE umur BETWEEN 25 AND 45
GROUP BY pekerjaan
ORDER BY rata_rata_pendapatan DESC
LIMIT 10;
```

## 3. Menyaring Hasil Agregasi (HAVING)

Ini adalah bagian yang sering membingungkan pemula. Apa bedanya `WHERE` dan `HAVING`?

- **WHERE:** Menyaring **data mentah** sebelum dihitung (seperti membuang baris yang tidak perlu).
- **HAVING:** Menyaring **hasil perhitungan** (seperti memfilter hasil Pivot Table).

### Contoh 1: HAVING Sederhana

```sql
-- Skenario: Tampilkan wilayah yang rata-rata pendapatannya di atas 12 juta
SELECT
    wilayah,
    COUNT(*) AS jumlah,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan
FROM responden
GROUP BY wilayah
HAVING AVG(pendapatan) > 12000000
ORDER BY rata_rata_pendapatan DESC;
```

### Contoh 2: HAVING dengan Filter Sampel Minimal

```sql
-- Skenario: Tampilkan pekerjaan yang memiliki minimal 10 responden
SELECT
    pekerjaan,
    COUNT(*) AS jumlah_responden,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan
FROM responden
GROUP BY pekerjaan
HAVING COUNT(*) >= 10
ORDER BY jumlah_responden DESC;
```

### Contoh 3: WHERE + GROUP BY + HAVING (3 Lapis Filter)

```sql
-- Skenario: Untuk responden usia 30+, tampilkan wilayah dengan total pendapatan > 500 juta
SELECT
    wilayah,
    COUNT(*) AS jumlah,
    ROUND(SUM(pendapatan), 0) AS total_pendapatan
FROM responden
WHERE umur >= 30  -- Filter data mentah dulu
GROUP BY wilayah
HAVING SUM(pendapatan) > 500000000  -- Filter hasil agregasi
ORDER BY total_pendapatan DESC;
```

## 4. Kombinasi: WHERE + GROUP BY + HAVING + ORDER BY

Dalam riset nyata, kita sering menggabungkan semua perintah yang sudah dipelajari.

### Contoh 1: Kompleks - 4 Lapis Operasi

```sql
-- Skenario: Di wilayah tertentu, tampilkan pekerjaan dengan avg pendapatan tertinggi
-- yang memiliki minimal 5 responden. Urutkan dari yang paling menguntungkan.
SELECT
    pekerjaan,
    wilayah,
    COUNT(*) AS jumlah,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan
FROM responden
WHERE wilayah IN ('Jakarta', 'Bandung', 'Surabaya')  -- Filter data mentah
GROUP BY pekerjaan, wilayah                           -- Kelompokkan 2 kolom
HAVING COUNT(*) >= 5                                  -- Filter hasil minimal 5 orang
ORDER BY rata_rata_pendapatan DESC                    -- Urutkan tertinggi
LIMIT 15;
```

### Contoh 2: Analisis Demografi Responden

```sql
-- Skenario: Tampilkan profil responden per status pernikahan
SELECT
    CASE WHEN status_pernikahan THEN 'Menikah' ELSE 'Belum Menikah' END AS status,
    COUNT(*) AS jumlah,
    ROUND(AVG(umur), 1) AS rata_rata_umur,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan
FROM responden
GROUP BY status_pernikahan
ORDER BY jumlah DESC;
```

### Contoh 3: Analisis Tren Pendapatan (Top Database Insights)

```sql
-- Skenario: Temukan wilayah-pekerjaan combo dengan pendapatan tertinggi
SELECT
    wilayah,
    pekerjaan,
    COUNT(*) AS jumlah_responden,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan,
    ROUND(MIN(pendapatan), 0) AS pendapatan_min,
    ROUND(MAX(pendapatan), 0) AS pendapatan_max
FROM responden
GROUP BY wilayah, pekerjaan
HAVING COUNT(*) >= 3  -- Minimal 3 orang di combo itu
ORDER BY rata_rata_pendapatan DESC
LIMIT 10;
```

## 5. Catatan: Query Execution Order (Penting!)

Saat menulis query dengan agregasi, database **selalu** mengeksekusi dalam urutan ini:

1. **FROM** — Baca tabel sumber (`responden`)
2. **WHERE** — Selesaikan filter data mentah (sebelum pengelompokan)
3. **GROUP BY** — Kelompokkan baris berdasarkan kolom yang dipilih
4. **HAVING** — Selesaikan filter grup (setelah pengelompokan)
5. **SELECT** — Hitung agregasi dan buat kolom hasil
6. **ORDER BY** — Urutkan hasil
7. **LIMIT** — Batasi jumlah baris yang ditampilkan

**Konsekuensi penting:**

- `WHERE` selalu SEBELUM `GROUP BY` → Filter individu dulu
- `HAVING` selalu SETELAH `GROUP BY` → Filter grup saja
- Alias kolom (AS) baru bisa digunakan di `ORDER BY`, tapi tidak di `WHERE` atau `HAVING`

## Latihan Mandiri Sesi 4

**Instruksi:** Tulis query SQL untuk setiap soal. Bantuan query struktur tersedia di bawah.

1. **Deskripsi Sampel:** Hitung total responden, rata-rata umur, umur termuda, dan umur tertua di seluruh database.

2. **Analisis Wilayah:** Tampilkan daftar wilayah dan hitung berapa banyak responden di masing-masing wilayah, lalu urutkan dari yang terbanyak.

3. **Analisis Pekerjaan:** Tampilkan top 10 kategori pekerjaan berdasarkan rata-rata pendapatan (dari yang paling tinggi).

4. **Filter Kompleks:** Tampilkan pekerjaan yang memiliki rata-rata pendapatan di atas 15 juta dan minimal 8 responden di wilayah Jakarta saja.

5. **Demografi Responden:** Analisis responden berdasarkan status pernikahan. Tampilkan jumlah, rata-rata umur, dan rata-rata pendapatan untuk setiap status.

6. **Kombinasi 2 Dimensi:** Buat ringkasan data per (wilayah + status pernikahan), tampilkan jumlah responden dan rata-rata pendapatan. Filter hanya grup dengan minimal 5 responden.

7. **Kategori Unik:** Berapa total kategori pekerjaan unik di database? Berapa rata-rata jumlah responden per pekerjaan?

8. **Analisis Umur-Pendapatan:** Buat grup responden berdasarkan range umur (18-25, 26-35, 36-45, 46-55, 56+). Untuk setiap grup, tampilkan jumlah dan rata-rata pendapatan.

### Kunci Jawaban Singkat (Opsional Cek Mandiri)

```sql
-- 1
SELECT
    COUNT(*) AS total,
    ROUND(AVG(umur), 1) AS rata_rata_umur,
    MIN(umur) AS umur_termuda,
    MAX(umur) AS umur_tertua
FROM responden;

-- 2
SELECT wilayah, COUNT(*) AS jumlah
FROM responden
GROUP BY wilayah
ORDER BY jumlah DESC;

-- 3
SELECT pekerjaan, ROUND(AVG(pendapatan), 0) AS rata_rata
FROM responden
GROUP BY pekerjaan
ORDER BY rata_rata DESC
LIMIT 10;

-- 4
SELECT pekerjaan, COUNT(*) AS jumlah, ROUND(AVG(pendapatan), 0) AS rata_rata
FROM responden
WHERE wilayah = 'Jakarta'
GROUP BY pekerjaan
HAVING AVG(pendapatan) > 15000000 AND COUNT(*) >= 8
ORDER BY rata_rata DESC;

-- 5
SELECT
    CASE WHEN status_pernikahan THEN 'Menikah' ELSE 'Belum Menikah' END,
    COUNT(*) AS jumlah,
    ROUND(AVG(umur), 1) AS rata_rata_umur,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan
FROM responden
GROUP BY status_pernikahan;

-- 6
SELECT wilayah, status_pernikahan, COUNT(*) AS jumlah, ROUND(AVG(pendapatan), 0) AS rata_rata
FROM responden
GROUP BY wilayah, status_pernikahan
HAVING COUNT(*) >= 5
ORDER BY wilayah;

-- 7
SELECT COUNT(DISTINCT pekerjaan) AS total_kategori,
       ROUND(AVG(responden_per_pekerjaan), 0) AS rata_rata_responden_per_pekerjaan
FROM (SELECT pekerjaan, COUNT(*) AS responden_per_pekerjaan FROM responden GROUP BY pekerjaan) AS subq;

-- 8
SELECT
    CASE
        WHEN umur BETWEEN 18 AND 25 THEN '18-25'
        WHEN umur BETWEEN 26 AND 35 THEN '26-35'
        WHEN umur BETWEEN 36 AND 45 THEN '36-45'
        WHEN umur BETWEEN 46 AND 55 THEN '46-55'
        ELSE '56+'
    END AS range_umur,
    COUNT(*) AS jumlah,
    ROUND(AVG(pendapatan), 0) AS rata_rata_pendapatan
FROM responden
GROUP BY range_umur
ORDER BY range_umur;
```

## Catatan Penting

**COUNT(\*) vs COUNT(kolom):**

- `COUNT(*)`: Menghitung SEMUA baris, termasuk baris dengan nilai NULL.
- `COUNT(kolom)`: Menghitung hanya baris yang memiliki nilai (exclude NULL).

Dalam dataset, umumnya tidak ada NULL, jadi keduanya sering menghasilkan hasil sama. Tapi penting untuk membedakannya di praktik riil.

**COUNT(DISTINCT) untuk Kategori Unik:**
Jika Anda ingin tahu berapa banyak _keunikan_ dalam 1 kolom, gunakan `COUNT(DISTINCT)`.

### Tips:

_"Jika Anda menggunakan fungsi agregasi (seperti `AVG` atau `COUNT`) bersamaan dengan kolom biasa (seperti `wilayah`), maka kolom biasa tersebut **WAJIB** masuk ke dalam `GROUP BY`. Jika tidak, PostgreSQL akan memberikan pesan eror karena ia bingung bagaimana cara menampilkan satu nilai rata-rata bersandingan dengan banyak nama wilayah."_

_"Urutan query agregasi penting: `SELECT` → `FROM` → `WHERE` (filter data mentah) → `GROUP BY` → `HAVING` (filter hasil) → `ORDER BY` → `LIMIT`. Tidak boleh dibolak-balik!"_
